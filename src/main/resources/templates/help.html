Help & Strategy Guide
How the bot decides, sizes, and manages each trade
What the bot does

TraderEdge auto-trades the NIFTY 50 universe using Central Pivot Range (CPR) level breakouts on a configurable intraday timeframe (default 5-min). It scans every watchlist symbol on every candle close, fires when a CPR / R / S / day-high / day-low level is broken with confirming conditions, then sizes and manages the trade through SL, partial target, and trailing.

How it picks stocks (the watchlist)

Each morning the bot pulls the NSE bhavcopy and computes today's CPR for every NIFTY 50 stock. Width as a % of price determines two universes:

  • Narrow CPR — width within [narrowCprMinWidth, narrowCprMaxWidth) (default 0% to 0.1%). Coiled CPR, breakout-friendly.
  • Inside CPR — today's CPR fully contained inside yesterday's CPR (lower-than-yesterday-volatility setup).

Each stock is then tagged Small or Large based on how wild yesterday was: prev-day range as % of 20-day ADR. Small ≤ 50% (compressed, coiled spring); Large > 50% (expanded, possibly exhausted). You toggle which combos to trade in Settings: NS, NL, IS, IL.

How it decides direction

For every 5-min candle close, the scanner walks through CPR/R/S levels in priority order:

Buys (green candle): R4 → R3 → R2 → R1+PDH → CPR → S1+PDL (magnet bounce) → Day High.
Sells (red candle): S4 → S3 → S2 → S1+PDL → CPR → R1+PDH (magnet rejection) → Day Low.

The first level that meets the breakout pattern wins. The level is then marked as "broken" so it can't re-fire today.

How it classifies the trade (LTF + HTF)

Every signal that survives is classified HPT (full size) or MPT (medium size, default 0.75) based on two timeframes:

  • LTF — 5-min breakout candle close vs daily CPR. Above daily TC = LTF bullish; below daily BC = LTF bearish; inside CPR = LTF undetermined.
  • HTF — weekly state machine (BULLISH / STRONG_BULLISH / BEARISH / STRONG_BEARISH).

Both align with trade direction → HPT. Only one aligns → MPT. Magnet setups (S1+PDL bounce, R1+PDH rejection) bypass the LTF gate and always classify as MPT. Counter-LTF non-magnet signals are rejected outright — there is no LPT under the new model.

How it confirms the trade

Before placing, the signal must pass a chain of independent filters. Any failure rejects the trade outright (no qty downgrade). Big filters in order:

  • ATP / VWAP confirmation (price vs day's VWAP)
  • 5-min SMA Price (lenient: vs 20+50; strict: vs 20+50+200)
  • 5-min SMA Alignment (lenient: 20 vs 50; strict: 20 vs 50 vs 200)
  • HTF Hurdle — prior 1h close above (buy) / below (sell) nearest weekly hurdle
  • HTF SMA Price + HTF SMA Alignment (60-min)
  • NIFTY trend filter (CPR + 5-min SMA Price + SMA 20/50 alignment — all 3 factors aligned)
  • Volume, candle-size, R/R, min-profit, day-type extended-level skips

See the PROBABILITY tab for the classification rule and the SETTINGS tab for every toggle.

Risk philosophy: choppy days small, trending days large

Under the LTF-priority model the bot is binary: either the conditions support the trade (HPT or MPT) or they don't (rejected). There's no LPT consolation prize. A clean trade fires at full size (HPT); a single-timeframe-aligned trade fires at medium size (MPT, default × 0.75); anything that fails LTF, HTF, or any of the gate filters is skipped outright.

Signal flow

The bot has one signal path (live, internal). At each scanner-timeframe candle close:

  1. CandleAggregator finalizes the just-closed bar from WebSocket ticks; SmaService / HtfSmaService / AtrService update.
  2. BreakoutScanner runs against every watchlist symbol — applies pre-filters (green/red candle, ATP, 5-min SMA Price/Alignment, volume), walks setup priority (R4 → ... → S1+PDL / R1+PDH magnets → DH/DL), runs the LTF check (close vs daily CPR) and HTF check (weekly state) to assign probability HPT / MPT / null. NIFTY alignment + 20-SMA level-count filters also run here. Counter-LTF non-magnet trades are rejected immediately.
  3. SignalProcessor applies the higher-level filter chain: DH/DL inside-zone gate, extended-level skips (IV/OV vs EV split for daily R3/S3, R4/S4; HTF weekly R3/S3, R4/S4), small/large candle, OR rules (EV days only), HTF Hurdle, HTF SMA Price & Alignment, target shift, R/R, qty stack, min profit. Failure rejects the trade outright — no qty downgrade.
  4. OrderService places the entry on Fyers. On fill, the structural SL and split targets (T1 + T2) are computed from the fill price and placed.
  5. OrderEventService tracks entry / SL / target lifecycle via the Order WebSocket. MarketDataService trails the SL via the tick stream once the Fibonacci stage triggers.

Two WebSockets, zero polling (in steady state)

  • Market data WS (binary HSM, wss://socket.fyers.in/hsm/v1-5/prod) — LTP, VWAP, volume. Feeds CandleAggregator and the scanner card SSE stream.
  • Order update WS (JSON, wss://socket.fyers.in/trade/v3) — entry fills, SL/target hits, manual modifications, external closes. Replaces 2-second polling.

If either WS drops, PollingService resumes 10-second REST polling automatically. Status shown in the sidebar pill.

Live UI

Stock cards push tick updates over Server-Sent Events (SSE) every market tick — LTP, change %, candle OHL, and SMAs (5-min and 1h, blended with current LTP) are all live. Full card data refreshes every 15 seconds (CPR levels, signal status, narrow/inside tags). NIFTY card LTP / change % update tick-by-tick; the cached three factors (CPR, SMA Price, SMA Align) refresh only on NIFTY's 5-min candle close.

Trade lifecycle

1. Entry

Once the signal passes all filters, OrderService places a market order on Fyers. The order ID is registered with OrderEventService for WS tracking; PollingService also watches it as a 10-second fallback.

2. Fill detection

When Fyers fills the order, the order WebSocket pushes a status update. The bot reads the actual fill price (not the candle close used for signal) and recomputes SL using the fill price. SL and split targets (T1 + T2) are placed immediately.

3. SL + split target placement

SL is a stop-market order. T1 and T2 are limit orders for a fraction each of the position. Default: T1 takes t1DistancePct% of the (T2 − entry) distance — e.g., 50% means T1 sits at the midpoint. T1 quantity = t1QtyPct% of the entry quantity. The remaining sits at T2.

4. T1 hit

When T1 fills, the SL is re-placed for the remaining quantity at the original SL price. (Optional behaviour: SL moves to entry / breakeven if configured.) T2 stays in place.

5. Trailing SL (Fibonacci-based)

If enableTrailingSl is on, MarketDataService monitors LTP. When LTP reaches fibStage1TriggerPct% (default 61.8%) of the (target − entry) range, SL trails up using fibStage1SlAtrMult × ATR from entry. At fibStage2TriggerPct% (default 78.6%), SL moves higher by fibStage2SlPct% of the range. Each trail modifies the existing SL order on Fyers, no cancel-and-replace.

6. Exit

Either T2 hits, SL hits, trailing SL hits, manual squareoff, or auto-squareoff time. The bot detects via order WebSocket, cancels the counterpart OCO leg, records the trade with P&L, clears position state, releases the level (so it can re-fire later if the conditions reset).

Manual interventions

  • Manual SL/Target modify — change in Fyers app or via Positions page; OrderEventService detects the modification within seconds and updates internal state.
  • Manual squareoff — Positions page button; bot cancels OCO and exits at market.
  • External close — if you close in Fyers app directly, bot detects via WS and clears state.
Failure handling

WebSocket disconnect

If either WS drops, the bot reconnects automatically. During the gap, PollingService picks up entry monitoring (every 2s) and OCO monitoring (every 5s). Status pill turns yellow ("POLLING").

Order rejected by Fyers

Logged with full Fyers reason. Position state never created. No retry — investigate root cause (margin, frozen scrip, circuit, etc.).

Mid-day restart

On startup, the bot loads cached positions from store/data/positions/ (one JSON per symbol), reconnects WSs, syncs against Fyers position book to reconcile. ATR, 5-min SMA, 1h SMA, and weekly CPR all restored from disk cache so no warm-up wait.

Auto squareoff

If autoSquareOffTime is set (e.g., "15:15"), all open positions are exited at market at that time. SL and target orders are cancelled. Single Telegram summary message sent.

IV / OV / EV — set by the first candle close

The bot classifies the day's "opening character" using the close of the first candle (default 5-min, so 9:20 close). This single tag drives several downstream decisions.

TagWhere the first candle closedMeaning
IVInside Value — inside daily CPR (between BC and TC)Quiet open inside today's pivot range. Normal trend day setup.
OVOutside Value — between CPR and R2/S2Modest gap; trend possible but watch for early reversal.
EVExtended Value — at or beyond R2 (gap up) or S2 (gap down)Strong gap. Trend-direction trades wait for OR confirmation.

What each tag enables / blocks

  • Extended-level skips (R3/S3, R4/S4) split toggles: separate IV/OV vs EV settings (skipR3S3IvOvDays, skipR3S3EvDays, skipR4S4IvOvDays, skipR4S4EvDays). Tighter on regular days, looser on gap days where extended-level breaks can be fresh momentum rather than exhaustion.
  • EV days use Opening Range gates: trades fire only after OR locks; direction must match OR break or align with the gap (see OPENING RANGE tab).
  • IV / OV days ignore OR entirely. Day-type also feeds NS/IS classification for the watchlist.
  • Counter-trend setups are day-type agnostic — both the 2 magnets (S1+PDL / R1+PDH) and the 6 deep mean-rev (BUY_ABOVE_S2/S3/S4, SELL_BELOW_R2/R3/R4) fire on any day type. All 8 are gated by the master toggle enableMeanReversionTrades; see the MEAN REVERSION tab.

Where you see it

Each scanner card shows the day open and tag, e.g. 1855.20 (IV). The tag is computed once when the first candle closes and remains fixed for the day.

Two breakout patterns per level

The candle qualifies via either:

  • Standard breakout — open or low (for buy) was below the level, close ended above. Direct break-through.
  • Wick rejection — open above level, low dipped below, close above. Bears tested support, bulls defended.

Sell is the mirror.

Scanner pre-filters (run before setup detection)

Before walking setup priority, the scanner applies these fast checks. Any failure short-circuits the whole symbol for that candle.

  • Green/red candle — buys need green, sells need red. Dojis skipped.
  • ATP / VWAP check (enableAtpCheck) — buy needs close above ATP; sell below.
  • 5-min SMA Price — lenient (enableSmaTrendCheckLenient): close above/below SMA 20 & 50. Strict (enableSmaTrendCheck): close above/below SMA 20, 50, 200 together. Strict implies lenient.
  • 5-min SMA Alignment — lenient (enableSmaAlignmentCheckLenient): 20 vs 50 ordering. Strict (enableSmaAlignmentCheck): 20 > 50 > 200 (buy) / 20 < 50 < 200 (sell). Strict implies lenient.
  • SMA vs ATP (enableSmaVsAtpCheck) — buy needs 20 SMA > ATP; sell < ATP.
  • SMA level-count filter (enableSmaLevelCountFilter) — skip if any CPR zone sits between 20 SMA and the broken level. Plus a proximity check (smaLevelMinRangePct). Optional morning skip bypass before smaLevelFilterMorningSkipUntil (default 10:15 IST).
  • Volume filter (enableVolumeFilter) — candle volume must be ≥ volumeMultiple × N-candle average.
  • NIFTY index alignment (enableIndexAlignment) — buys require NIFTY state == BULLISH; sells require BEARISH. SIDEWAYS / NEUTRAL / opposite-direction → reject. No soft mode.
  • LTF check — non-magnet trades require breakout close above daily TC (buy) / below daily BC (sell). Otherwise rejected as LTF_OPPOSED.
  • Probability assignment — HPT (LTF + HTF aligned), MPT (only one aligned, or magnet trade), null (rejected).
Magnet trades — bounce off support / rejection at resistance

Magnet setups are short-distance mean-reversion plays toward the central pivot. Two of them:

  • BUY_ABOVE_S1_PDL — bounce off S1+PDL zone, target CPR top.
  • SELL_BELOW_R1_PDH — rejection at R1+PDH zone, target CPR bottom.

Why the probability rule is different

A magnet is by definition a counter-trend bounce on the daily timeframe — buying support in a stock currently below daily TC, or selling resistance in a stock above daily BC. The LTF check (close vs daily CPR) would always fail by design. So magnets bypass the LTF gate entirely and are sized as MPT regardless of LTF/HTF state.

SetupClassification
BUY_ABOVE_S1_PDLMPT — always (LTF gate bypassed; HTF state ignored)
SELL_BELOW_R1_PDHMPT — always

Magnets still pass through the rest of the filter chain (HTF Hurdle, HTF SMA, NIFTY align, OR rules, candle filters, etc.). Failure in any of those rejects the trade outright.

Targets

Magnet targets aim back to the value area:

  • BUY_ABOVE_S1_PDL → T1 = min(TC, BC) (CPR bottom), T2 = min(R1, PDH).
  • SELL_BELOW_R1_PDH → T1 = max(TC, BC) (CPR top), T2 = max(S1, PDL).

Quantity

MPT qty factor (mptQtyFactor, default 0.75) applies. Other multipliers stack on top (session move limit, etc.) — see the QTY STACKING tab.

Six setups that fire bounces from deep support and fades from deep resistance. Counter-LTF by design: like magnets, they bypass the LTF gate at probability assignment and classify as MPT. Day-type agnostic — fire on IV / OV / EV alike, as long as the candle pattern matches.

Priority order

Within the buy detection chain, S4 is checked first, then S3, then S2 — deepest reclaimed level wins. Same idea for sells (R4 > R3 > R2). The 6 mean-rev setups sit between the S1+PDL magnet (higher priority) and DH (lowest priority).

Master toggle

The setting enableMeanReversionTrades (default on) is a master switch for all 10 counter-trend setups: the 2 magnets, the 6 deep mean-rev, and the 2 DH/DL day-extreme breakouts. When off, all ten skip with bucket MEAN_REVERSION_DISABLED in Signal Trail. Only direction-aligned weekly-level breakouts (R1+/R2/R3/R4 buys, S1+/S2/S3/S4 sells) continue firing.

Quantity

Sized by the MPT factor (mptQtyFactor, default 0.75) only. Both the EV-reversal halving and the OV-magnet halving were removed — counter-trend setups (the 8 magnets + mean-rev) get one consistent qty reduction via MPT and no setup-specific extras.

Opening Range (OR)

OR = the high & low of the first openingRangeMinutes minutes of the session (default 15 min, so 9:15 to 9:30). At 9:30 the OR locks and becomes a fixed reference for the rest of the day.

What OR does (EV days only)

  • OR not yet locked → no EV-specific gating yet (signals still go through the regular filter chain).
  • OR broken, trade direction matches the break → allowed.
  • OR broken, trade direction opposes the break → rejected.
  • Close inside OR, direction aligns with the gap → allowed.
  • Close inside OR, direction opposes the gap → rejected (no break, fighting the gap).

IV / OV days don't use OR for gating — the LTF check (close vs daily CPR) already enforces direction.

Target shift bypass when inside OR

Normally target gets shifted to the nearest weekly level / session H-L between entry and structural target. When the entry candle closes inside the OR range, this shift is skipped — the OR boundary itself is the immediate overhead/underfoot structure, so capping target at session H-L would be misleading.

Weekly trend

Weekly trend uses 60-min HTF candles vs the weekly CPR levels. During the first hour of the session the reference is live LTP (no completed 1h candle yet); after 10:15 it switches to the last completed 1h close.

Inside-CPR refinement (live)

When price sits inside the weekly CPR (between BC and TC), the bot doesn't blindly call it NEUTRAL. It checks live LTP vs HTF SMA 20 & 50:

  • LTP > both 1h SMAs → BULLISH (early read of which side is winning)
  • LTP < both 1h SMAs → BEARISH
  • LTP between the two SMAs → truly NEUTRAL

This refinement uses live LTP throughout the day — outside-CPR is sticky (1h close), inside-CPR is a live probe.

Daily trend

Same logic as weekly but on daily CPR (from yesterday's bhavcopy). Reference price = last completed 5-min close (LTP fallback for first 5-min). Tags: STRONG_BULLISH / BULLISH / NEUTRAL / BEARISH / STRONG_BEARISH using daily R1/PDH, S1/PDL, TC, BC.

How it gates trades

When enableIndexAlignment is on, every signal direction is checked against NIFTY's state. Buys allowed when NIFTY = BULLISH or BULLISH_REVERSAL; sells allowed when NIFTY = BEARISH or BEARISH_REVERSAL. Every other state (SIDEWAYS / NEUTRAL / opposite-direction) skips the trade outright. The reversal states are intentionally treated as valid entry conditions — when CPR's bias has been overruled by both SMA factors, the structural turn often anticipates the trade direction. No soft mode, no qty reduction.

Why this combination: CPR sets the day's directional bias from yesterday's close; the 5-min SMA pair captures whether intraday price action confirms that bias. Demanding all three to align means we're trading the index alongside both its longer and shorter timeframes pointing the same way. Mixed signals → wait it out.

HPT vs MPT (LTF-priority model)

Every signal that fires is classified as either HPT (full size, qty × 1.0) or MPT (medium size, qty × mptQtyFactor, default 0.75). The classification is decided by two timeframes — the LTF and the HTF. Counter-LTF signals are rejected outright, not classified. LPT no longer exists for new trades.

The two timeframes

  • LTF = 5-min breakout candle's close vs daily CPR. Above daily TC = LTF bullish; below daily BC = LTF bearish; inside CPR = LTF undetermined.
  • HTF = weekly state machine output (BULLISH / STRONG_BULLISH for the bull side; BEARISH / STRONG_BEARISH for the bear side).

Classification rule

Setup categoryLTF alignedHTF alignedTier
Standard breakout (CPR, R1+PDH, R2/R3/R4, S1+PDL, S2/S3/S4, DH/DL)YesYesHPT
Standard breakoutYesNoMPT
Standard breakoutNo / undetermined—Rejected (LTF_OPPOSED)
Magnet (BUY_ABOVE_S1_PDL, SELL_BELOW_R1_PDH)(bypassed)(bypassed)MPT always

Magnets bypass both gates because they're counter-trend bounce/rejection setups by design — they buy support in stocks below daily TC and sell resistance in stocks above daily BC, so the LTF gate would always fail. They fire as MPT and rely on the rest of the filter chain for safety.

Filter chain — every failure is a reject

HPT and MPT trades both pass through the same downstream filter chain. Any failure rejects the trade outright (no qty downgrade, no LPT consolation). Order roughly matches execution order — scanner-side first, then SignalProcessor-side:

Scanner-side (BreakoutScanner — runs before HPT/MPT classification)

  1. Green/red candle — buys need green, sells need red; dojis skipped.
  2. ATP / VWAP (enableAtpCheck) — buy needs close > ATP; sell < ATP.
  3. 5-min SMA Price — lenient (enableSmaTrendCheckLenient): close above/below SMA 20 & 50. Strict (enableSmaTrendCheck): close above/below SMA 20, 50, 200 together.
  4. 5-min SMA Alignment — lenient (enableSmaAlignmentCheckLenient): 20 vs 50. Strict (enableSmaAlignmentCheck): 20 > 50 > 200 (buy) / 20 < 50 < 200 (sell).
  5. SMA vs ATP (enableSmaVsAtpCheck) — buy needs 20 SMA > ATP; sell < ATP.
  6. 20-SMA level-count (enableSmaLevelCountFilter) — skip if a CPR zone sits between the 20 SMA and the broken level; plus proximity check (smaLevelMinRangePct).
  7. Volume filter (enableVolumeFilter) — candle volume ≥ volumeMultiple × N-candle average.
  8. NIFTY index alignment (enableIndexAlignment) — buys allowed when NIFTY = BULLISH or BULLISH_REVERSAL; sells allowed when NIFTY = BEARISH or BEARISH_REVERSAL; SIDEWAYS / NEUTRAL / opposite skip.
  9. NIFTY HTF Hurdle (enableNiftyHtfHurdleFilter, opt-in) — same rule as the per-stock HTF Hurdle, applied to NIFTY's own data. NIFTY's prior 1h close must have decisively cleared NIFTY's nearest weekly hurdle in trade direction (R1/PWH/weekly TC/Pivot/BC for buys; S1/PWL/... for sells). While NIFTY's current 1h candle is in progress and the prior close is on the wrong side, all stock signals skip.
  10. HTF Candle Direction (enableHtfCandleFilter, opt-in) — per-stock candle-direction confluence on the in-progress 60-min HTF bar. Buy requires 1h close > 1h open (green); sell requires 1h close < 1h open (red). Doji (open == close) passes both. Fail-open if no 1h bar formed yet for the symbol.

SignalProcessor-side (after probability assignment)

  1. Extended-level skips — daily R3/S3, R4/S4 (per day-type IV/OV vs EV) and HTF weekly R3/S3, R4/S4.
  2. OR rules on EV days — direction must match the OR break, or align with the gap if no break yet.
  3. HTF Hurdle — prior 1h close above (buy) / below (sell) the nearest weekly hurdle (R1, PWH, weekly TC, Pivot, BC for buys; mirror for sells).
  4. HTF SMA Price — prior 1h close above / below 1h SMA 20 and 50 together.
  5. HTF SMA Alignment — 1h SMA 20 vs 50 ordering matches direction.
  6. Small / Large candle filters — body + move-past-level floors; large-body cap as multiple of ATR.
  7. Session move limit — qty halved if already moved > sessionMoveLimit% from open / PDC.
  8. Target shift, target tolerance, structural SL, R/R filter, min absolute profit.

Probability toggles

enableHpt and enableMpt independently disable each tier. With enableMpt = false, only HPT signals fire (cleaner setups, fewer trades). The legacy enableLpt setting is preserved in RiskSettingsStore for backward-compat config loading and historical Trade Log display only — no new trade is ever classified or downgraded to LPT.

Quantity impact

HPT = full base qty (× 1.0). MPT = base qty × mptQtyFactor (default 0.75 → 25% reduction). Other multipliers stack on top (session move limit) — see the QTY STACKING tab.

What is the Signal Trail

The Signal Trail is a post-mortem audit of every breakout signal the scanner saw today. Open it from the scanner page (the link next to the candle countdown chip). For each candle close on every watchlist symbol it captures one row:

  • TRADED — the signal passed all filters and an order was placed.
  • FILTERED — the signal was rejected. The row records the structured filter name (the bucket) and a one-line detail explaining why.
  • ERROR — exception thrown during processing (rare; investigate via Console logs).

The header row shows total counts and the top-blockers as clickable chips — click a chip to filter the table to that bucket.

Practical use
  • End-of-day review — scroll the trail looking for symbols with many filtered rows in one bucket. If, e.g., HTF_SMA_TREND dominates today's blocks, the 1h SMA bundle was misaligned for most stocks — that's a market-context observation, not necessarily a bug.
  • Tuning — if a setup repeatedly fails one bucket and you trust the rest of the chain, consider relaxing that gate (e.g. switch SMA Price from strict to lenient). Don't tune off a single day — accumulate a few sessions first.
  • Debugging missed trades — found a chart where the move was obvious but no trade fired? Find the symbol+time in the Signal Trail. The bucket and detail tell you exactly which gate skipped it.
  • Filter chip filtering — the top-of-modal chips show the day's distribution of buckets. Click one to scope the table to that bucket only.
Quantity stacking order

Quantity is computed in steps; each multiplier stacks on the previous. The order matters because each step rounds to even (min 2 shares).

  1. Base qty — risk-based when fixedQuantity = -1: baseQty = riskPerTrade / |entry − SL|, capped by capital × leverage. Otherwise the fixed number is used (rounded to even, min 2).
  2. Probability factor — HPT × 1.0; MPT × mptQtyFactor (default 0.75). Counter-LTF non-magnet trades never reach this step (rejected upstream).
  3. Session move limit (enableSessionMoveLimit) — if price has already moved more than sessionMoveLimit% from day open or previous day close, qty is halved. Catches over-extended setups.
  4. Minimum absolute profit gate (minAbsoluteProfit) — expected profit = qty × |target − entry|. If less than threshold, the trade is rejected outright. Catches setups where R/R is fine but absolute rupees are too small.

Worked example

Stock at 1000, ATR 5, structural SL at 992 (8 below entry). riskPerTrade = 1000, so base qty = 1000 / 8 = 125 → rounded to 124 (even).

  • If weekly disagrees with LTF (MPT): 124 × 0.75 = 93 → 92.
  • If also session moved 2.3% (above 2% limit): 92 / 2 = 46.
  • Final: 46 shares. Expected profit (target +12 from entry): 46 × 12 = ₹552. If minAbsoluteProfit = 500, allowed.

The stack is intentionally compounding — partial alignment + extended price action shrinks size, fully aligned setups trade full HPT size.

Split logic

When enableSplitTarget is on and target distance ≥ splitMinDistanceAtr × ATR, the position is split:

  • T1 sits at t1DistancePct% of (T2 − entry) distance — default 50%, so the midpoint between entry and T2.
  • T1 quantity is t1QtyPct% of total qty (e.g., 50%).
  • T2 takes the remaining qty.

What happens on T1 hit

  1. Partial fill detected via Order WebSocket.
  2. SL re-placed for the remaining qty at the original SL price (not breakeven by default).
  3. T2 stays in place and continues to monitor.
  4. P&L for the partial recorded immediately, full close logged when T2 fills.

Target shifting (before split)

Before placing T1/T2, the structural target may be shifted closer to entry if a real S/R level sits between them:

  • Session high/low shift (enableDayHighLowTargetShift) — shift to today's session body H/L if it's within the path. Bypassed when entry candle is inside OR.
  • Weekly level shift (enableWeeklyLevelTargetShift) — pool: weekly R1/R2/R3/R4, S1/S2/S3/S4, PH, PL. Excludes weekly TC/BC/Pivot (pivot range interior, not real S/R).
  • Weekly SMA shift (enableWeeklySmaTargetShift) — shift to 1h SMA 20/50/200 if between entry and target.

The nearest qualifying level wins. Rationale: if there's a wall between you and the structural target, target there instead — better to take a smaller win than have the trade reverse at the wall.

Target tolerance

Real prices rarely tap exactly at a CPR level — they nudge close, reverse, then resume. enableTargetTolerance + targetToleranceAtr (default 0.10 × ATR) discounts the structural target by a small ATR fraction so near-miss reversals still fill. e.g., if R2 = 1010 and ATR = 4, target is placed at 1010 − 0.40 = 1009.60. Buys that get within 40 paise of R2 still take their win.

Stop loss calculation

Default (ATR-based)

SL = close ± atrMultiplier × ATR (default multiplier 1.5). Used when enableStructuralSl is off.

Structural SL (recommended)

When enableStructuralSl is on, the SL anchors to the broken level / candle wick rather than just the entry close:

  • Zone setups (CPR, R1+PDH, S1+PDL, magnets) — anchor = far edge of the zone (e.g., min(TC, BC) for buy-above-CPR). Zone width itself acts as a built-in cushion.
  • Single-level setups (R2/R3/R4, S2/S3/S4) — anchor = the level itself.
  • DH/DL setups — anchor = the breakout candle's own low (buy) or high (sell). The candle's wick is the closest structural defence.

SL = anchor ± (structuralSlBufferAtr + extra) × ATR. Default base buffer = 1.0 × ATR.

Single-level extra buffer

Single-level setups (R2+, S2+, DH/DL) get an additional singleLevelSlBufferAtr × ATR (default 0.5) added to the base buffer. Reason: zone setups have built-in zone-width cushion; single-level setups don't, so SL gets nicked too easily on routine retests. Default total: zone = 1.0 × ATR, single-level = 1.5 × ATR from anchor.

Trailing SL (Fibonacci-based)

When enableTrailingSl is on, MarketDataService monitors LTP. As LTP advances toward the target:

  • Stage 1 at fibStage1TriggerPct% of (target − entry) range (default 61.8%) — SL trails to entry ± fibStage1SlAtrMult × ATR (default 1.0). For a buy this typically locks in breakeven plus a small buffer.
  • Stage 2 at fibStage2TriggerPct% (default 78.6%) — SL moves to entry + fibStage2SlPct% of range (default 61.8%). Locks in most of the move.

The SL order is modified in place on Fyers (no cancel-replace). Trailing only ratchets in the trade's favour; it never moves backward.

Risk / reward filter + defensive guards

After SL and target are computed, two safety checks run before the trade is sized:

  1. Wrong-side target guard — for buy: target must be above entry. For sell: target below entry. Catches degenerate target projections (e.g., projected R5 below current price after a runaway DH break). Trade rejected if target is on the wrong side.
  2. R/R filter (enableRiskRewardFilter) — reject if reward / risk < minRiskRewardRatio (default 1.0:1). Catches setups where SL is too wide relative to target distance.

Filter "kill switches"

Most filters can be turned off independently from the Settings page. Useful when testing or in unusual market conditions:

  • Disable enableSmaTrendCheck / enableSmaTrendCheckLenient to allow trades against the SMA bundle. Same for the alignment pair.
  • Disable enableHtfHurdleFilter, enableHtfSmaAlignment, enableHtfSmaAlignmentCheck independently.
  • Disable enableIndexAlignment to ignore NIFTY's state entirely.
  • Disable enableMpt to trade HPT-only (only fully-aligned LTF + HTF setups fire — cleanest, fewest trades).
  • Disable enableHpt for the rare case of MPT-only experimentation.
